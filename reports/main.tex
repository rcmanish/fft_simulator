       and Hybrid Parallelisation with Signal Verification}

\begin{abstract}
This paper presents a comprehensive, high-performance C++ implementation of the
Discrete Fourier Transform (DFT) and Cooley-Tukey Fast Fourier Transform (FFT).
We rigorously verify numerical correctness via Parseval's theorem, IFFT round-trip
errors, and DFT-versus-FFT agreement across seven representative signal types,
with all errors below $10^{-12}$.
Seven simulation experiments validate physical phenomena including Fourier duality,
Lorentzian lineshapes, two-tone resolution, SNR sensitivity, 2D transform properties,
and wave-equation modal energy conservation.
OpenMP parallelisation achieves $11.9\times$ speedup at 16 threads (74.6\% efficiency)
for embarrassingly parallel batch FFTs (Strategy~A), while single-FFT parallelisation
(Strategy~B) is limited to $2.3\times$ by barrier synchronisation between
$\log_2 N$ butterfly stages.
MPI parallelisation with an Alltoall transpose achieves linear reduction of per-rank
compute time at large $N$, with communication growing as $O(\log P)$.
The hybrid $4{\times}4$ configuration achieves the best throughput
($14.2\times$ over serial), reducing MPI Alltoall message count while saturating
shared-memory bandwidth.
\end{abstract}

\section{Introduction}
\IEEEPARstart{T}{he} Fourier transform is the mathematical foundation of signal
processing, PDE solvers, and image analysis~\cite{oppenheim1999,brigham1988}.
The na\"ive DFT scales as $\mathcal{O}(N^2)$; the Cooley-Tukey FFT~\cite{cooley1965}
reduces this to $\mathcal{O}(N\log N)$.
State-of-the-art libraries such as FFTW~\cite{frigo2005} further reduce constant
factors via cache-oblivious recursive decomposition.
This paper documents a complete, verified FFT pipeline and explores shared-memory
(OpenMP~\cite{openmp2018}), distributed-memory (MPI~\cite{mpi2021}),
and hybrid parallelisation strategies.

\section{Mathematical Formulation}
The forward DFT of a discrete signal $x[n]$, $n\in[0,N-1]$, is:
\begin{equation}
X[k] = \sum_{n=0}^{N-1} x[n]\,e^{-i 2\pi kn/N},\quad k\in[0,N-1].
\label{eq:dft}
\end{equation}
The IDFT is $x[n]=\frac{1}{N}\sum_{k}X[k]e^{+i2\pi kn/N}$.
Parseval's theorem states:
\begin{equation}
\sum_{n}|x[n]|^2 = \frac{1}{N}\sum_{k}|X[k]|^2.
\label{eq:parseval}
\end{equation}
The radix-2 Cooley-Tukey algorithm decomposes an $N$-point DFT into two $N/2$-point
DFTs via the butterfly operation, achieving $\mathcal{O}(N\log_2 N)$
complexity~\cite{cooley1965}.
Window functions reduce spectral leakage~\cite{harris1978};
Welch's method reduces PSD variance~\cite{welch1967}.

\section{Serial Implementation}
The core uses \texttt{std::vector<std::complex<double>>} with in-place bit-reversal
followed by $\log_2 N$ butterfly stages.

\section{OpenMP Parallelisation}
We explore four distinct parallelisation axes~\cite{golub2013}:
\begin{enumerate}
  \item \textbf{Strategy A (Batch 1D)}: Parallel outer loop over independent FFT
        instances. Achieves $11.9\times$ at 16 threads (74.6\% efficiency) for
        8192 FFTs of size $N=4096$. Near-ideal scaling because the workload is
        embarrassingly parallel.
  \item \textbf{Strategy B (Single large FFT)}: Parallel butterfly stages of a
        single $N=2^{22}$ FFT. Achieves only $2.3\times$ at 16 threads because
        $\log_2 N=22$ barrier synchronisations between stages each cost $\sim\!100\,\mu$s,
        totalling $\sim\!2.2$ ms of pure overhead for a $\sim\!10$ ms kernel.
  \item \textbf{Strategy C (STFT)}: Parallel STFT frames.
        Achieves $11.8\times$ at 16 threads (73.6\% efficiency) as frame count
        provides sufficient parallelism for most signal lengths.
  \item \textbf{Strategy D (2D FFT)}: Parallel row-FFTs, then column-FFTs.
        Achieves $9.6\times$ at 16 threads (60\% efficiency) because the
        column pass accesses strided memory, degrading cache utilisation.
\end{enumerate}

\section{MPI Parallelisation}
For distributed memory, we implement a 1D Cooley-Tukey FFT with
\texttt{MPI\_Alltoall} transpose~\cite{mpi2021}.
Rank $i$ holds elements $[iN/P,\,(i+1)N/P)$.
Local butterfly stages compute $\log_2(N/P)$ levels;
\texttt{MPI\_Alltoall} then redistributes for the remaining $\log_2 P$ global levels.

Compute time (blue) decreases monotonically as $1/P$; Alltoall communication (orange)

\centering
\includegraphics[width=0.9\columnwidth]{../plots/mpi_strong_1d.pdf}

Fig.~\ref{fig:mpi_comm} illustrates the key result: at large $N$, per-rank compute

At small $N$ (as in earlier runs), communication dominates immediately, producing
flat or increasing apparent compute time --- confirming that scaling experiments must
use sufficiently large $N$.

because Alltoall dominates at this batch size relative to the compute benefit.

\section{Discussion}

\subsection{Comparison with FFTW}
FFTW~\cite{frigo2005} achieves near-peak hardware performance through a runtime planner
that selects the optimal algorithm for each $N$ and cache configuration.
For $N=2^{22}$ on a modern 16-core processor, FFTW achieves $\sim\!15$ GFLOP/s,
$\approx3\times$ higher than our implementation, primarily due to its cache-oblivious
recursive decomposition.

\subsection{Cache-Oblivious Algorithms}
Our iterative Cooley-Tukey implementation accesses twiddle factors in a strided
pattern during the late butterfly stages, causing L3 cache misses at $N>2^{20}$
(Fig.~\ref{fig:cache}).
A cache-oblivious FFT~\cite{frigo2005} rearranges computation so sub-problems of
size $\sqrt{N}$ fit in cache at each recursive level, eliminating the performance cliff.

\subsection{NUMA Topology}
For the hybrid configuration, NUMA effects become significant beyond one socket.
Threads accessing remote-socket memory incur $\approx2\times$ higher latency.
Optimal hybrid placement keeps all OMP threads within one NUMA domain (typically
8--16 cores per socket), consistent with the winning $4{\times}4$ configuration.

\section{Conclusions}
We designed, verified, and parallelised a complete Cooley-Tukey FFT pipeline.
Parseval's theorem is preserved to $<10^{-12}$ across all seven signal types.
Seven experiments validate Fourier duality, Lorentzian lineshapes, spectral resolution,
SNR sensitivity, 2D transforms, and wave-equation modal energy conservation.
OpenMP Strategy~A achieves $11.9\times$ speedup at 16 threads (74.6\% efficiency);
Strategy~B is limited to $2.3\times$ by butterfly-stage barriers.
MPI achieves linear per-rank compute reduction at $N=2^{24}$ with Alltoall
communication growing as $O(\log P)$.
The hybrid $4{\times}4$ configuration achieves the highest throughput ($14.2\times$),
demonstrating the advantage of reducing MPI communication overhead while exploiting
shared-memory bandwidth.
